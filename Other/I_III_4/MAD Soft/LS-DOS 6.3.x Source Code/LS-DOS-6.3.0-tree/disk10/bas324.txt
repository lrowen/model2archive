              BAS3TO4 filespec1 filespec2 (m,c,p,w=nn)Where:  filespec1 is a compressed (not saved in ASCII) BASIC program file to be translated from Model 3 BASIC to Model 4 BASIC. If afilespec is not given on the command line, the prompt "Input filespec?"will appear and the filespec may be entered from the keyboard.  filespec2 is the file to receive the translated program, which willbe an ASCII file with most of the necessary changes to run under Model 4BASIC.  If filespec2 is not given on the command line, the prompt"Output filespec?" will appear and the filespec may be entered from thekeyboard.parameters:   MODIFY    adjust PRINT@ statements with numeric arguments to the value             required to print at the same location on the Model 4 screen             (relative to the top left corner).   CENTER    add an additional offset to that calculated by the MODIFY             parameter to allow centering on the Model 4 screen. If CENTER             is specified with no value, the default value of 328 (4 lines             plus 8 characters) will be added to each numeric PRINT@ argument             after the value is calculated for the MODIFY parameter. If the             form CENTER=nnnn is used for the parameter, a numeric value may             be given for the additional offset. When both MODIFY and CENTER             are specified, PRINT TAB (nn) statements are also adjusted by the             relative offset per line determined from the CENTER value             (the default value would be 8).   PRINT     print a list of the line numbers and statements that may need             additional manual correction to operate as expected under Model 4             BASIC.  If PRINT is not specified, this list is displayed on the             video.   WIDTH     specifies the maximum line length for the listing of the             statements that may need additional corrections.  If WIDTH is             not specified, the default value of 80 characters per line will             be used.  Values from 10 to 255 may be specified for WIDTHAll parameters may be abbreviated to their first character.The program in the input file will be translated as necessary to load intoModel 4 BASIC and written to the output file.  A list of the statements thatare not directly compatable between the two versions of BASIC will bedisplayed on the video or printer. This list may be used to modify thesestatements to operate under Model 4 BASIC. The following statements are listed as possible problems when encountered inthe Model 3 program:RESETSETCMDIF (unless followed by THEN)OUTNAMESYSTEMPOKEPRINT@PRINT TABPRINT #-1, PRINT #-2INPUT #-1, INPUT #-2CLOADCSAVEUSRERRPOINTTIME$INPPOSPEEKPRINT@ statements are listed in the form PRINT@(EXP) when the argumentis an expression or variable.  If the MODIFY parameter is not specified,the form PRINT@(nnnn) is used when the argument is numeric. If MODIFY isspecified, the form PRINT@(aaa=>bbb) is used, where aaa is the number inthe Model 3 program, and bbb is the number as modified for the Model 4 screen.TAB statements are also listed with arguments in a similar form if the MODIFYand CENTER parameters are specified.The program modifications may cause the acceptable line length for Model 4BASIC to be exceeded.  In this case, the line will be truncated at a lengthwhich will allow the program to be loaded into BASIC, and the deleted portionof the line will be displayed in the listing of incompatible statements.  Itwill generally be necessary to add an additional program line to re-insertthis portion of the program, taking care not to change the program logic ifthe line contains an IF statement.
